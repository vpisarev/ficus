# Ficus design log — сессия 06–07 июля 2026

Компактный журнал решений для переноса контекста. Артефакты: brief #1
(ficus_test_infra_brief.md), brief #2 (ficus_harden_brief_2.md), found_bugs.md
(в репо, реестр Opus).

## Стратегическое

- Решение: Фикус пилим. Позиционирование (вынести в README/статью):
  «Fortran/Matlab по духу, нулевой футпринт, быстрее C++ за счёт loop fusion,
  безопасный». Целевая персона: вычислительный инженер.
- Killer app: фоторедактор «телефонного стиля» (слайдеры + NN-автоулучшатели +
  поиск + коллажи), Linux. Промежуточная цель: CLI RAW-процессор (даст тесты
  для будущего UI). Архитектура: движок на Ficus как C-библиотека, GUI отдельно.
- C — основной бэкенд навсегда (прозрачность цен операций, ABI, вечность);
  `-c++` — опция для интеропа. К loops добавить C-интерфейс.
- AI-стратегия: Ficus доказанно AI-легибелен (Opus за сессию вышел на свободное
  письмо: 68K строк корпуса + быстрый цикл ошибок компилятора). CLAUDE.md
  собирать из самоотчётов агента после каждой крупной сессии («обо что
  спотыкался в первый час»); критерий качества: свежая сессия с CLAUDE.md
  разгоняется быстрее, чем без него.
- Python-стратегия: `ficus --pymodule` — генерация CPython-расширений
  (zero-copy через buffer protocol, abi3-wheels, GIL отпускать, прицел на
  free-threading). Приоритет высокий, но ПОСЛЕ реформы синтаксиса (внешние
  пользователи удорожают breaking changes). pip = пакетный менеджер Фикуса
  (прецеденты: mojo, cmake). Отдельного package manager не будет.
- loops (github.com/4ekmah/loops): кандидат в JIT-субстрат (intrinsics
  gemm/conv, REPL-бэкенд). Для REPL не хватает одной фичи: косвенный CALL
  (через указатель); трогает RA (caller-saved spill) — brief с ревью аллокатора.
  RVV в loops планируется (без grouping в v1, m1-only). REPL — по остаточному
  принципу.

## Решения по языку (спека для brief #3)

### fold — императивная форма (решено)
- Тело fold = void-выражение; аккумуляторы — скоупированные переменные,
  обновляются присваиванием по имени. Родословная: Fortran do concurrent
  reduce, Chapel forall with reduce.
- Кортежное присваивание `(a, b) = (b, a+b)` — одновременная семантика
  (критично для автомигратора: старые тела-кортежи = одновременные обновления;
  наивное a=b; b=a+b неверно, п. Фибоначчи). Временные переменные тоже ок.
- break/continue легальны. Warning только если аккумулятор НЕ упоминается в
  теле вообще; подавление `acc = _`. Per-branch warning НЕ делать (шум).
- Вложенный fold может присваивать аккумуляторам внешнего (обычные правила
  скоупов/затенения). Захват аккумулятора замыканием = обычный захват var
  (в K-форме аккумуляторы уже var; fold — сахар, испаряется до K-формы).
- Миграция громкая (старые тела типа кортеж vs требуемый void — всё падает
  ошибкой типов). Критерий приёмки: K-форма мигрированного кода идентична
  побитово (T4-снапшоты).
- @parallel у общего fold НЕДОПУСТИМ (осознанно). Вместо него — именованные
  редукционные сахара: sum, minmax (один проход!), argmin/argmax (редукция
  значение+индекс), count, mean, sumabs/sumsqr/maxabs (нормы). Источник
  списка — API OpenCV. Гистограмма (indexed reduction/scatter-add) — не v1,
  но дизайн набора не должен закрывать к ней дверь. user-defined reduction
  (identity, combine) — далёкий v2. Распознавание редукций по K-форме — если
  делать, то на стабильной K0 + жёсткая ошибка при неудаче под @parallel
  (никаких тихих откатов в последовательность).
- Проверить: fold внутри array comprehension `[fold s=0 for x<-xs {…}]` как
  бесплатный scan (префиксные суммы) — легален ли, что даёт.

### Generics — направление list[t] (пробуем)
- `list[t]` вместо `'t list`. Двусмысленность с индексацией только в
  выражениях; решение по образцу Go/Nim: семантическое (голова = тип или
  значение), либо v1 вообще без явной инстанциации в выражениях (вывод +
  аннотации, как OCaml).
- Тип и его автогенерированные функции-конструкторы = одна сущность при
  разрешении (Point[float] как тип и как вызов — одна инстанциация).
- Корпусный скрипт: посчитать затенения тип/значение в 68K строк (ожидание ~0).
- Бонус: result[a, b] лечит многопараметрическую беду ML-нотации и открывает
  путь к нормальной частичной специализации.
- Методика реформ вообще: измерять на корпусе (сколько мест меняется,
  неоднозначности парсера), рендерить один код в N вариантах синтаксиса для
  A/B глазами, все breaking одной эпохой + автомигратор (AST-переписыватель).

### Семантика (спека)
- `>>`: арифметический для знаковых, логический для беззнаковых. Записать в
  туториал (сейчас нигде).
- ConstInt в фолдере: (int64, bool), bool=true только для uint64 (решение
  Вадима, минимум правок). Аудит div/mod/cmp/прочих ConstInt-веток.
- FB-008 стратегия: per-module (стабильные ключи сортировки на эмиссии →
  per-module счётчики манглинга; per-module id только при недостаточности).
  Есть de-mangling pass (C_post_rename_locals), локальные чистые, резидуал —
  глобальный уровень + порядок деклараций (Вадим помнит проблемы с ним).
  content-based mangling — в roadmap (инкрементальная компиляция).
